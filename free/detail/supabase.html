关于 Supabase

Supabase 是一个开源的 Firebase 替代品，提供完整的后端解决方案，包括 PostgreSQL 数据库、身份验证、存储、边缘函数等功能。其免费计划为开发者提供了一个低门槛体验企业级后端服务的机会，非常适合个人项目、MVP开发和学习使用。

开源 PostgreSQL 后端即服务 实时数据库

免费计划特点

数据库存储

提供 500 MB 的 PostgreSQL 数据库存储空间，支持完整的 SQL 功能和实时订阅。

包含所有PostgreSQL功能

文件存储

提供 1 GB 的文件存储容量，可用于存储用户上传的图片、文档等各类文件。

支持直接访问和CDN

带宽限制

每天提供 5 GB 的出站流量（不含缓存），满足小型应用的流量需求。

每日重置配额

活跃用户

每月最多支持 50,000 名活跃用户，适合用户量不大的应用和原型项目。

无强制用户上限

边缘函数

每天最多 1,000 次调用，每秒最多 2 次调用，适用于轻量级后端逻辑。

有调用限制

项目数量

每个组织最多可以创建 2 个免费项目，适合同时开发多个小型应用。

限2个项目

注意：免费计划适用于非商业用途，如用于商业项目请遵守 Supabase 使用条款。所有功能均无需信用卡即可使用。

关键限制点

项目数量限制

每个组织最多只能创建 2 个免费项目，超出需要升级到付费计划或创建新组织。

项目自动暂停

如果项目连续 7 天未使用，系统将自动暂停该项目，需要手动重新激活。

功能限制

免费计划不支持自定义域名、自动备份、日志保留超过 1 天等高级功能。

边缘函数限制

每个项目最多 100 个边缘函数，最大内存 256MB，最大执行时间 150 秒。

邮件发送限制

每月最多发送 1,000 封邮件，超出需使用自定义 SMTP 服务（如 SendGrid 或 Mailgun）。

1 访问官网

2 创建账户

使用 GitHub 账号、Google 账号或邮箱地址注册一个 Supabase 账户，无需信用卡。

GitHub Google 邮箱
3

3 创建项目

登录后点击"New Project"按钮，按照以下步骤配置：

  • 选择组织（个人账号默认有一个个人组织）
  • 填写项目名称（英文推荐）
  • 设置强密码（用于数据库访问）
  • 选择地区（建议选择离目标用户近的地区）
  • 选择"Free Plan"
  • 点击"Create Project"按钮

激活技巧与优化建议

保持项目活跃

定期访问项目或设置自动化任务，避免 7 天未使用导致项目被自动暂停。可利用 GitHub Actions 设置每周访问检查。

合理规划资源

在免费计划限制内进行开发，优化数据库查询，压缩存储文件，避免超出配额导致服务中断。

使用自定义 SMTP 服务

当需要发送大量邮件时，配置外部 SMTP 服务（如 SendGrid 免费计划）绕过每月 1,000 封的限制。

监控使用情况

通过 Dashboard 定期查看资源使用情况，设置使用阈值提醒，及时调整资源消耗较大的功能。

利用社区支持

在 GitHub Discussions 或 Discord 社区寻求帮助，这是免费计划获取技术支持的主要渠道。

多环境策略

利用 2 个免费项目配额，分别创建开发和生产环境，避免开发测试影响生产数据。
