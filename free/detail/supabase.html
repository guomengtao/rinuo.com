What is Supabase?

Supabase is an open-source backend-as-a-service (BaaS) that provides all the backend features you need to build a product. It's built on top of PostgreSQL and offers a suite of tools that make it easy to develop modern applications.

Think of it as an alternative to Firebase, but with a PostgreSQL database instead of a NoSQL solution. Supabase aims to provide developers with the tools they need to build full-stack applications without having to worry about setting up and maintaining complex backend infrastructure.

The project was started in 2020 and has since grown into a popular choice for developers looking for an open-source alternative to proprietary BaaS solutions.

Project Origin

Supabase was founded in 2020 by Paul Copplestone and Ant Wilson. The idea emerged from their frustration with the lack of open-source alternatives to Firebase.

They believed that developers should have access to powerful backend tools without being locked into proprietary systems. PostgreSQL, being open-source and feature-rich, seemed like the perfect foundation.

The project started as a weekend experiment but quickly gained traction in the developer community. By 2021, Supabase had raised seed funding and established a dedicated team to build out the platform.

Today, Supabase is used by thousands of developers worldwide and continues to grow as a viable open-source alternative to closed backend services.

Pros & Cons

Advantages

  • Open-source with no vendor lock-in
  • Powered by PostgreSQL, a robust relational database
  • Realtime updates without writing complex backend code
  • Generous free tier for development and small projects
  • Easy to self-host if needed
  • Excellent documentation and community support

Disadvantages

  • Less mature than Firebase or AWS Amplify
  • Some advanced features are still in development
  • Requires some PostgreSQL knowledge for complex queries
  • Self-hosting requires more setup compared to managed services
  • Smaller ecosystem of plugins and extensions
  • Limited geographic regions for hosted service

Getting Started with Supabase

Free to Use

Supabase offers a generous free tier that's perfect for development and small production projects. You can sign up for free without a credit card.

Step 1: Create an Account

Head over to supabase.com and sign up for a free account using your email or GitHub account.

Step 2: Create a Project

Once logged in, click "New Project" and follow the setup wizard. You'll need to provide a project name, password, and select a region.

Step 3: Explore the Dashboard

Familiarize yourself with the dashboard where you can manage your database, authentication, storage, and other services.

Step 4: Integrate with Your App

Install the Supabase client library in your project and connect using your project's API keys, which you can find in the dashboard settings. 

npm install @supabase/supabase-js
