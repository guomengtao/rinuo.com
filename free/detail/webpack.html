What is Webpack?

Webpack is a powerful static module bundler for modern JavaScript applications. It takes modules with dependencies and generates static assets representing those modules.

At its core, Webpack analyzes your project's structure, finds JavaScript modules and other assets, and packages them into one or more bundles that can be loaded by the browser. It's highly configurable and can handle not just JavaScript but also HTML, CSS, images, and more.

Webpack helps developers manage complex codebases, optimize assets for production, and improve loading performance through techniques like code splitting, lazy loading, and asset optimization.

How to Use Webpack

Terminal

# Install Webpack

npm install webpack webpack-cli --save-dev

# Create a basic configuration file

touch webpack.config.js

# Run Webpack in development mode

npx webpack --mode development

# Run Webpack in production mode (optimized)

npx webpack --mode production

See it in Action!

Here's a basic webpack.config.js example:

const path = require('path');

module.exports = {
  entry: './src/index.js',
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader'],
      },
      {
        test: /\.(png|svg|jpg|jpeg|gif)$/i,
        type: 'asset/resource',
      },
    ],
  },
};
