Docker 免费服务

Docker 提供了丰富的免费工具和服务，帮助开发者构建、测试和部署容器化应用

Docker Engine

Docker 核心引擎完全免费开源，基于 Apache 2.0 许可证，可在任何操作系统上运行。

Docker Compose

用于定义和运行多容器 Docker 应用程序的工具，完全免费，简化了复杂应用的部署。

Docker Hub 免费账户

提供免费的公共仓库，可托管无限的公共镜像和有限数量的私有镜像，方便分享和协作。

官方文档与教程

详尽的官方文档、指南和教程完全免费，帮助开发者快速掌握 Docker 的使用和最佳实践。

社区支持

活跃的社区论坛、GitHub 讨论和 Stack Overflow 问答，免费获取技术支持和解决方案。

Docker SDKs

多种编程语言的软件开发工具包完全免费，方便开发者通过代码与 Docker 引擎交互。

Docker 的优点

Docker 凭借容器化技术彻底改变了软件的开发、测试和部署方式

环境一致性

容器包含应用运行所需的所有依赖，确保开发、测试和生产环境完全一致，解决"在我机器上能运行"的问题。

资源高效利用

容器共享主机操作系统内核，比虚拟机更轻量，启动更快，占用资源更少，可在同一主机上运行更多应用。

快速部署与扩展

容器可以在几秒内启动，支持快速扩展和缩减，非常适合微服务架构和持续集成/持续部署(CI/CD)流程。

隔离性与安全性

容器之间相互隔离，一个容器的问题不会影响其他容器，提供了一定程度的安全边界和资源控制。

开发者满意度

Docker 的缺点

尽管Docker有很多优势，但在某些场景下也存在一些局限性

学习曲线陡峭

对于新手来说，理解容器、镜像、卷、网络等概念需要一定时间，特别是在复杂部署场景下。

安全性考量

虽然提供隔离，但容器共享主机内核，与虚拟机相比隔离性较弱，需要额外措施确保安全。

存储管理复杂

容器的临时性本质使得数据持久化需要特殊处理，卷管理和数据备份可能变得复杂。

Windows支持有限

虽然有Windows容器，但生态系统主要围绕Linux，某些功能在Windows上支持不完善或性能较差。

Docker 使用方法

从安装到基本操作，快速入门 Docker

步骤 1: 安装 Docker

Docker 支持多种操作系统，以下是不同系统的安装方法：

Windows 安装

# 推荐使用 Docker Desktop for Windows
# 下载地址: https://www.docker.com/products/docker-desktop

# 安装完成后验证
docker --version
docker run hello-world

Windows 10 专业版/企业版需要开启 Hyper-V，家庭版会使用 WSL 2 后端

macOS 安装

# 下载 Docker Desktop for Mac
# 下载地址: https://www.docker.com/products/docker-desktop

# 安装完成后验证
docker --version
docker run hello-world

需要 macOS 10.15 或更高版本，支持 Intel 和 Apple Silicon 芯片

Linux 安装 (Ubuntu 示例)

# 更新 apt 包索引并安装依赖
sudo apt-get update
sudo apt-get install ca-certificates curl gnupg lsb-release

# 添加 Docker 官方 GPG 密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 设置稳定版仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker 引擎
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io

# 将当前用户添加到 docker 组，避免使用 sudo
sudo usermod -aG docker $USER

# 验证安装
docker --version
docker run hello-world

其他 Linux 发行版请参考 官方安装文档

步骤 2: Docker 基本命令

掌握这些基本命令，开始使用 Docker 管理容器和镜像：

镜像相关命令

# 从仓库拉取镜像
docker pull [镜像名]:[标签]
docker pull ubuntu:22.04
docker pull nginx:latest

# 列出本地镜像
docker images
docker image ls

# 删除本地镜像
docker rmi [镜像ID或名称]
docker image rm [镜像ID或名称]

# 搜索镜像
docker search [关键词]

容器相关命令

# 运行容器
docker run [选项] [镜像名] [命令]
docker run -it ubuntu:22.04 /bin/bash # 交互式运行Ubuntu
docker run -d -p 8080:80 nginx # 后台运行Nginx并映射端口

# 列出运行中的容器
docker ps
docker container ls

# 列出所有容器（包括停止的）
docker ps -a

# 启动、停止、重启容器
docker start [容器ID或名称]
docker stop [容器ID或名称]
docker restart [容器ID或名称]

# 进入运行中的容器
docker exec -it [容器ID或名称] /bin/bash

# 删除容器
docker rm [容器ID或名称]
docker container rm [容器ID或名称]

系统信息命令

# 查看Docker版本
docker --version
docker version

# 查看Docker系统信息
docker info

# 查看容器日志
docker logs [容器ID或名称]

# 查看容器详细信息
docker inspect [容器ID或名称]

步骤 3: 构建自定义镜像

通过 Dockerfile 构建自己的镜像，打包应用程序及其依赖：

创建 Dockerfile

在项目根目录创建名为 Dockerfile 的文件：

# 以官方Python镜像为基础
FROM python:3.9-slim

# 设置工作目录
WORKDIR /app

# 复制当前目录文件到容器的/app目录
COPY . .

# 安装依赖
RUN pip install --no-cache-dir -r requirements.txt

# 暴露端口
EXPOSE 5000

# 容器启动时执行的命令
CMD ["python", "app.py"]

构建镜像

# 在Dockerfile所在目录执行
docker build -t my-python-app:1.0 .

# -t 选项指定镜像名称和标签
# . 表示当前目录为构建上下文

# 查看构建的镜像
docker images | grep my-python-app

优化镜像构建

# 使用 .dockerignore 文件排除不必要的文件
# 创建 .dockerignore 文件
__pycache__
*.pyc
*.pyo
*.pyd
.env
.git
venv

# 多阶段构建减小镜像体积（示例）
FROM node:16 AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/build /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]

优化镜像可以减小体积、提高安全性并加快部署速度

步骤 4: 管理多容器应用

使用 Docker Compose 定义和运行多容器 Docker 应用程序：

创建 docker-compose.yml

version: '3.8'

services:
  web:
    build: .
    ports:
      - "5000:5000"
    depend_on:
      - db
    environment:
      - DATABASE_URL=postgresql://user:password@db:5432/mydb
    restart: always

  db:
    image: postgres:13
    volumes:
      - postgres_data:/var/lib/postgresql/data/
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=mydb
    restart: always

volumes:
  postgres_data:

使用 Docker Compose 命令

# 启动所有服务（后台运行）
docker-compose up -d

# 启动所有服务并查看日志
docker-compose up

# 停止并删除容器、网络
docker-compose down

# 停止并删除容器、网络、镜像和卷
docker-compose down --rmi all -v

# 查看服务状态
docker-compose ps

# 查看服务日志
docker-compose logs
docker-compose logs web # 只查看web服务日志
docker-compose logs -f # 实时查看日志

# 构建或重新构建服务
docker-compose build
docker-compose build web # 只构建web服务

容器网络与数据持久化

# 查看Docker网络
docker network ls
docker network inspect [网络名称]

# 创建自定义网络
docker network create my-network

# 查看卷
docker volume ls
docker volume inspect [卷名称]

# 创建卷
docker volume create my-volume

# 使用卷运行容器
docker run -d -v my-volume:/data --name my-container nginx

卷(volumes)是持久化容器数据的推荐方式，比绑定挂载更具可移植性

步骤 5: 部署与分享镜像

将你的镜像推送到仓库，与他人分享或部署到生产环境：

使用 Docker Hub

# 登录到Docker Hub
docker login

# 为镜像打标签（格式：用户名/仓库名:标签）
docker tag my-python-app:1.0 username/my-python-app:1.0
docker tag my-python-app:1.0 username/my-python-app:latest

# 推送到Docker Hub
docker push username/my-python-app:1.0
docker push username/my-python-app:latest

# 从Docker Hub拉取并运行
docker pull username/my-python-app:latest
docker run -d -p 5000:5000 username/my-python-app:latest

注册 Docker Hub 账号: https://hub.docker.com/

使用私有仓库

# 使用自托管或云服务商的私有仓库
# 登录到私有仓库
docker login my-private-registry.example.com

# 为镜像打标签
docker tag my-app:1.0 my-private-registry.example.com/my-app:1.0

# 推送到私有仓库
docker push my-private-registry.example.com/my-app:1.0

# 从私有仓库拉取
docker pull my-private-registry.example.com/my-app:1.0

常见的私有仓库解决方案包括 AWS ECR、Google GCR、Azure ACR 和自托管的 Harbor
