The Story Behind Insomnia

From a personal project to a developer favorite

Meet the Founder

Gregory Schier
Creator of Insomnia

Gregory is a software engineer with a passion for building tools that make developers' lives easier. With experience at companies like Netflix and Microsoft, he understood the pain points of API development firsthand.

How It All Started

Insomnia began in 2014 as a personal project when Gregory Schier got frustrated with the API tools available at the time. He wanted something simpler, more intuitive, and focused on the developer experience.

What started as a side project quickly gained traction in the developer community. By 2016, Insomnia had grown to thousands of users and had become a full-time endeavor for Gregory.

In 2019, Insomnia was acquired by Kong, a leading API platform company, allowing the project to grow even faster while maintaining its focus on building the best possible developer experience.

Today, Insomnia continues to evolve with regular updates, new features, and a thriving community of contributors and users around the world.

The Good and the Bad

An honest look at Insomnia's strengths and weaknesses

Advantages

  • Intuitive, user-friendly interface that's easy to learn
  • Excellent support for both REST and GraphQL APIs
  • Powerful scripting capabilities for automation
  • Great collaboration features for team environments
  • Free and open-source core with optional paid features
  • Regular updates and active development
  • Cross-platform support (Windows, macOS, Linux)

Disadvantages

  • Some advanced features require a paid subscription
  • Can be resource-intensive on lower-end machines
  • Learning curve for advanced scripting features
  • Limited offline documentation
  • Some protocol support is not as robust as specialized tools
  • Occasional performance issues with very large collections
  • Sync features can be slow with large teams
