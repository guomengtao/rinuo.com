名字含义与起源

CentOS 是什么意思？

CentOS 是 "Community Enterprise Operating System"（社区企业操作系统）的缩写。这个名字反映了它的本质：一个由社区支持的企业级操作系统。

起源与发展

CentOS 项目始于2004年，由社区开发者创建，旨在提供一个与Red Hat Enterprise Linux (RHEL) 二进制兼容的免费操作系统。

它通过重新编译RHEL的源代码（去除Red Hat的商标和专利软件）来实现这一目标，使企业和个人能够免费使用一个稳定、可靠的企业级Linux发行版。

2014年，Red Hat宣布收购CentOS项目，但承诺保持其开源性质和免费提供。2020年，Red Hat宣布CentOS Linux将逐步过渡到CentOS Stream，作为RHEL的上游开发分支。

长期支持

步骤 1: 下载CentOS镜像

访问CentOS官方网站或其镜像站点，下载适合您硬件架构的最新版本ISO镜像文件。

# 官方下载地址
https://www.centos.org/download/

推荐下载DVD版本，包含更多预装软件；如果您需要最小化安装，可以选择Minimal版本。

步骤 2: 制作启动盘

使用工具将下载的ISO文件写入USB闪存驱动器，制作可启动安装介质。

  • Windows: 使用Rufus或Etcher
  • macOS: 使用Etcher或命令行dd工具
  • Linux: 使用Etcher或命令行dd工具
# Linux/macOS命令行示例
sudo dd if=CentOS-Stream-9-x86_64-dvd1.iso of=/dev/sdX bs=4M status=progress

步骤 3: 安装系统

将启动盘插入计算机，从USB启动，按照安装向导的指示完成CentOS的安装。

  • 选择安装语言和键盘布局
  • 配置磁盘分区（可以使用自动分区）
  • 设置root密码和创建普通用户
  • 选择要安装的软件包组

安装过程可能需要几分钟到半小时，取决于硬件性能和所选软件包数量。

步骤 4: 基本配置

安装完成后，首次启动系统并进行基本配置。

# 更新系统
sudo dnf update -y

# 配置防火墙
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload

# 启用SELinux（可选）
sudo setenforce 1

步骤 5: 安装所需软件

根据您的需求，使用dnf包管理器安装所需的软件。

# 安装Web服务器
sudo dnf install httpd -y
sudo systemctl enable --now httpd

# 安装数据库服务器
sudo dnf install mariadb-server -y
sudo systemctl enable --now mariadb

# 安装开发工具
sudo dnf groupinstall "Development Tools" -y

可以通过dnf search命令查找需要的软件包，例如：sudo dnf search python

优缺点分析

客观评估CentOS的优势和不足，帮助您做出更合适的选择

优点

  • 免费使用，无许可费用，降低企业IT成本
  • 与RHEL完全兼容，可运行相同的应用程序
  • 长期支持周期，每个版本提供10年更新支持
  • 优秀的稳定性和安全性，适合生产环境
  • 丰富的软件仓库和包管理系统
  • 广泛的硬件支持和驱动程序
  • 庞大的用户社区和丰富的文档资源
  • 适合企业级应用和服务器部署

缺点

  • 软件版本相对较旧，更新速度较慢
  • 2020年后的战略转变引起社区担忧
  • CentOS Linux将停止更新，转向CentOS Stream
  • 对新手不够友好，需要一定的Linux知识
  • 桌面环境支持有限，主要面向服务器
  • 某些专有软件可能不提供CentOS版本
  • 重大版本升级需要重新安装系统
  • 企业级支持需要付费购买Red Hat服务
