What is Helm?

Helm is a package manager for Kubernetes that simplifies deploying and managing applications and services on Kubernetes clusters.

Think of Helm as the apt, yum, or homebrew for Kubernetes. It allows you to package Kubernetes resources into a single unit called a "Chart" that can be versioned, shared, and deployed.

Helm helps you manage Kubernetes applications by defining, installing, and upgrading even the most complex Kubernetes deployments.

Project Origin & Founders

The Story Behind Helm

Helm was created in 2015 by Deis (now part of Microsoft) as an open-source project to address the complexity of managing Kubernetes applications.

The original creators recognized that Kubernetes YAML files could become complex and difficult to manage, especially for larger applications. They set out to create a tool that would simplify this process through templating and versioning.

In 2019, Helm became a graduated project under the Cloud Native Computing Foundation (CNCF), joining other important cloud-native technologies like Kubernetes itself, Prometheus, and Envoy.

Technology Stack

Core Technologies

  • Go Programming Language

    Helm is primarily written in Go, making it lightweight and efficient.

  • Kubernetes API

    Deep integration with Kubernetes API for managing resources.

  • YAML/JSON

    For configuration files and Kubernetes manifest definitions.

  • Mustache Templates

    Used for Helm chart templating to create dynamic Kubernetes manifests.

  • OCI Registries

    For storing and distributing Helm charts (Helm 3+).

Advantages & Disadvantages

Advantages

  • Simplified Deployment

    Reduces complex Kubernetes YAML configurations to simple commands.

  • Version Control for Deployments

    Track and manage different versions of your application deployments.

  • Reusability

    Create reusable charts that can be shared across teams and projects.

  • Rollbacks Made Easy

    Quickly roll back to previous versions of your deployments with a single command.

  • Large Ecosystem

    Access to thousands of pre-built charts in public repositories like Artifact Hub.

Disadvantages

  • Learning Curve

    Requires understanding of both Helm and Kubernetes concepts.

  • Chart Maintenance

    Charts need to be updated as Kubernetes versions and best practices evolve.

  • Security Concerns

    Public charts may contain vulnerabilities if not properly vetted.

  • State Management

    Can be challenging to manage stateful applications compared to stateless ones.

  • Helm 2 to 3 Migration

    Breaking changes required significant migration effort for existing users.

Install Helm Now

Quick Start Guide

1. Install Helm

# For macOS
brew install helm

# For Windows (Chocolatey)
choco install helm

# For Linux
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

2. Add a Chart Repository

helm repo add bitnami https://charts.bitnami.com/bitnami

3. Search for Charts

helm search repo bitnami

4. Install a Chart

helm install my-release bitnami/nginx

5. See Your Releases

helm list

6. Upgrade a Release

helm upgrade my-release bitnami/nginx
