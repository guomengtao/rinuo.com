什么是 Vercel？

Vercel 是一个云平台即服务 (PaaS)，专为前端开发者设计，专注于静态网站和 Jamstack 应用的部署。由 Next.js 的创建者开发，Vercel 提供了无缝的部署体验，特别适合 React、Vue、Svelte 等现代前端框架。

Vercel 的核心理念是"开发者体验至上"，通过简化部署流程、提供全球 CDN 分发和自动预览等功能，让开发者能够专注于代码而非基础设施。

Vercel 的优缺点

优点

  • 部署流程简单

    连接代码仓库后自动部署，无需复杂配置

  • 全球 CDN 网络

    内容分发速度快，全球用户访问延迟低

  • 与 Git 工作流无缝集成

    自动为每个 PR 创建预览，简化团队协作

  • 对 Next.js 完美支持

    作为 Next.js 创造者，提供最佳运行环境

  • 慷慨的免费计划

    免费计划包含足够资源供个人开发者使用

  • 优秀的开发者体验

    直观的控制台、详细的文档和良好的 CLI 工具

  • 自动 HTTPS 和证书管理

    无需手动配置 SSL 证书，自动更新

缺点

  • 高级功能需付费

    许多高级功能如团队协作、高级分析等需要订阅付费计划

  • 对非 Next.js 框架支持有限

    虽然支持其他框架，但优化不如 Next.js 完善

  • 免费计划有资源限制

    Serverless 函数调用次数、带宽等有明确限制

  • 数据存储选项有限

    内置数据存储服务相对简单，复杂应用仍需外部数据库

  • 迁移成本较高

    使用了 Vercel 特有功能的项目迁移到其他平台较困难

  • 付费计划价格较高

    对于流量大或功能需求多的项目，成本可能较高

  • 某些地区访问速度一般

    在部分亚洲和非洲地区，访问速度不如欧美地区理想

Vercel 使用方法

快速入门指南，帮助你开始使用Vercel部署应用

步骤 1: 注册 Vercel 账号

首先，你需要注册一个 Vercel 账号。Vercel 提供多种注册方式，选择最适合你的一种：

通过 GitHub 注册（推荐）

  1. 访问 Vercel 官网
  2. 点击右上角的 "Sign Up" 按钮
  3. 选择 "Continue with GitHub" 选项
  4. 授权 Vercel 访问你的 GitHub 账号（需要至少有公共仓库访问权限）
  5. 完成简单的用户信息设置，即可创建账号

其他注册方式

除了 GitHub，Vercel 还支持以下注册方式：

  • GitLab 账号
  • Bitbucket 账号
  • 电子邮件注册

注册完成后，你会自动获得免费计划的所有权益，无需信用卡。免费计划对于个人开发者和小型项目已经足够使用。

步骤 2: 创建和部署项目

注册完成后，你可以开始创建和部署第一个项目。Vercel 支持多种项目创建方式：

从 GitHub 仓库部署（推荐）

  1. 在 Vercel 控制台点击 "New Project" 按钮
  2. 如果你已经连接了 GitHub，会看到你的仓库列表
  3. 选择你想要部署的仓库（如果看不到，可能需要在 GitHub 中授权更多仓库）
  4. 点击仓库名称进入部署配置页面

使用 Vercel 模板

Vercel 提供了多种框架的模板，适合快速开始新项目：

  1. 在 "New Project" 页面，滚动到 "Templates" 部分
  2. 选择合适的模板（如 Next.js、React、Vue 等）
  3. 点击 "Deploy" 按钮，Vercel 会自动创建仓库并部署

通过 CLI 部署

对于本地项目，可以使用 Vercel CLI 部署：

# 安装 Vercel CLI
npm i -g vercel

# 登录账号
vercel login

# 进入项目目录并部署
cd your-project
vercel

按照提示完成部署，首次部署会要求确认项目设置。

步骤 3: 配置部署选项

在部署项目前，你可以配置一些部署选项，以确保项目正确运行：

基本配置

  • 项目名称：默认为仓库名称，可自定义
  • 团队选择：个人账号或团队（免费计划只能使用个人账号）
  • 框架预设：Vercel 会自动检测框架，通常无需修改

构建和输出设置

对于需要构建的项目（如 React、Vue 等），需要配置：

# 常见框架的构建命令示例
Next.js: 无需配置（自动检测）
React: npm run build
Vue: npm run build
Angular: npm run build --prod
  • 构建命令：项目构建的命令
  • 输出目录：构建结果的目录（如 dist、build）
  • 安装命令：依赖安装命令（通常是 npm install）

环境变量

对于需要环境变量的项目（如 API 密钥、后端地址等）：

  1. 点击 "Environment Variables" 部分的 "Add" 按钮
  2. 输入变量名和值（如 API_KEY=your_key）
  3. 对于敏感信息，勾选 "Secret" 选项
  4. 可以为不同环境（开发、预览、生产）设置不同的值

完成部署

配置完成后，点击 "Deploy" 按钮开始部署过程：

  • Vercel 会自动克隆代码、安装依赖、构建项目
  • 部署完成后，会显示项目的 URL（通常是 project-name.vercel.app）
  • 点击 URL 即可访问你的应用

步骤 4: 配置自定义域名

免费计划支持添加自定义域名，让你的网站拥有专业的域名：

添加自定义域名

  1. 在项目控制台中，点击 "Settings" 选项卡
  2. 选择左侧菜单中的 "Domains"
  3. 在输入框中输入你的域名（如 example.com），点击 "Add"
  4. Vercel 会验证域名所有权并提供 DNS 配置说明

DNS 配置

根据你的域名注册商，需要添加以下 DNS 记录：

记录类型 主机名 值 TTL
A @ 76.76.21.21 600
CNAME www your-project.vercel.app 600

具体的 IP 地址可能会变化，请参考 Vercel 提供的最新配置信息。

HTTPS 配置

配置域名后，Vercel 会自动处理 HTTPS：

  • 自动生成和配置 SSL 证书
  • 默认启用 HTTP 到 HTTPS 的重定向
  • 证书会自动续期，无需手动管理

证书配置可能需要几分钟到几小时，请耐心等待 DNS 记录生效。

步骤 5: 探索进阶功能

Vercel 提供了许多进阶功能，可以提升你的开发和部署体验：

预览部署

利用 Vercel 的预览部署功能，在合并代码前测试更改：

  • 向你的仓库提交 Pull Request
  • Vercel 会自动创建一个预览部署
  • 在 PR 评论中会显示预览链接
  • 团队成员可以在合并前测试新功能

Serverless Functions

创建无服务器函数处理后端逻辑：

# 在项目根目录创建 api 文件夹
mkdir api

# 创建一个简单的 API 端点
# api/hello.js
export default function handler(req, res) {
  res.status(200).json({ message: 'Hello from Vercel!' })
}

部署后，可以通过以下 URL 访问：

https://your-project.vercel.app/api/hello

分析和监控

监控项目性能和使用情况：

  • 在项目控制台的 "Analytics" 查看访问统计
  • 查看函数执行时间和资源使用情况
  • 设置警报，当超出免费配额时收到通知
  • 分析页面性能和加载时间

集成和自动化

扩展 Vercel 功能的集成选项：

  • 与 Slack、Discord 集成，接收部署通知
  • 使用 Webhook 触发自定义工作流
  • 集成内容管理系统（CMS）如 Contentful、Sanity
  • 连接数据库服务如 Supabase、Firebase
