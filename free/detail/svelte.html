What is Svelte?

Unlike traditional frameworks that do most of their work in the browser, Svelte shifts that work into a compile step that happens when you build your app.

Instead of using techniques like virtual DOM diffing, Svelte writes code that surgically updates the DOM when the state of your app changes.

Creator & Project Origin

RH

Rich Harris

Svelte was created by Rich Harris, a British software developer who worked at The New York Times. He first conceived of Svelte while trying to solve performance problems in newsroom tools.

The first version of Svelte was released in 2016. Unlike React and Vue, which use a virtual DOM, Harris wanted a framework that compiled directly to efficient vanilla JavaScript.

Today, Rich continues to lead the development of Svelte alongside a dedicated team of open-source contributors.

Technical Stack

Svelte works great with these complementary technologies

SvelteKit

The official full-stack framework for Svelte, providing routing, server-side rendering, and more.

Prisma

A modern database toolkit that works seamlessly with Svelte for data modeling and access.

Tailwind CSS

A utility-first CSS framework that pairs perfectly with Svelte's component model.

Vercel / Netlify

Platforms for deploying Svelte applications with serverless functions and global CDN.

Cypress

An end-to-end testing framework that works great for testing Svelte applications.

GraphQL

A query language for APIs that integrates smoothly with Svelte applications.

Pros & Cons

An honest look at Svelte's strengths and weaknesses

Small Bundle Sizes

Svelte compiles to vanilla JS with no runtime, resulting in smaller bundle sizes.

Excellent Performance

No virtual DOM means faster updates and better runtime performance.

Simple Learning Curve

Familiar HTML, CSS, and JavaScript syntax makes Svelte easy to learn.

Built-in State Management

No need for external state libraries like Redux for most applications.

Smaller Ecosystem

Not as many libraries and tools as React or Vue.

Less Job Opportunities

Fewer companies using Svelte compared to more established frameworks.

Compile Step Required

Unlike vanilla JS, you need a build process to use Svelte.

Less Mature Tooling

Developer tools and IDE support aren't as mature as other frameworks.

How to Get Started with Svelte

1

Install Node.js

Svelte requires Node.js (v14 or later) to run the build tools.

2

Create a new project

Run npm create svelte@latest my-app in your terminal.

3

Install dependencies

Navigate to your project folder and run npm install.

4

Start development server

Run npm run dev to start the local development server.
