什么是 Expo？

Expo 是一个开源平台，用于通过 JavaScript 和 React 构建原生 iOS、Android 和 web 应用。

跨平台开发

一次编写，多平台运行。使用相同的代码库构建 iOS、Android 和 web 应用。

快速开发周期

通过热重载和 OTA 更新，快速迭代你的应用，无需等待冗长的原生构建过程。

丰富的 API

访问设备功能如相机、位置、通知等，无需编写原生代码。

名字的含义与起源

"Expo" 一词来源于 "Exposition"（博览会），象征着展示和汇聚各种技术和可能性的平台。

Expo 起源于 2015 年，由 James Ide、Brent Vatne 和 Bruce Wang 创立，当时名为 "Exponent"。 最初是作为 React Native 的一个工具集，旨在简化移动应用开发流程。

2017 年，项目更名为 "Expo"，并逐渐发展成为一个完整的平台，包括开发工具、SDK、构建服务和部署解决方案。

2015 - 至今

如何开始使用 Expo

1
安装 Node.js 和 Expo CLI

确保你的系统已安装 Node.js (推荐 v16 或更高版本)，然后安装 Expo CLI。

npm install -g expo-cli
2
创建新项目

使用 Expo CLI 创建一个新的 Expo 项目。

expo init my-project
3
启动开发服务器

进入项目目录并启动开发服务器。

cd my-project
expo start
4
预览应用

使用 Expo Go 应用扫描终端中的 QR 码，或在模拟器中运行应用。

Expo 的优缺点

了解 Expo 的优势和局限性，帮助你决定是否适合你的项目需求。

优点

简化的开发流程

无需配置复杂的原生开发环境，降低了移动应用开发的入门门槛。

快速迭代

热重载和 OTA 更新功能让开发者可以快速看到更改效果，加速开发周期。

丰富的 API

内置大量 API 可直接访问设备功能，无需编写原生代码。

跨平台一致性

在 iOS、Android 和 web 上提供一致的开发体验和应用行为。

强大的社区支持

活跃的开发者社区和丰富的第三方库，解决问题更加容易。

缺点

应用体积较大

Expo 应用通常比纯原生应用或纯 React Native 应用体积更大。

原生模块限制

不能直接使用所有 React Native 原生模块，需要 eject 才能使用某些模块。

依赖 Expo 团队

某些功能更新依赖于 Expo 团队的开发进度，可能滞后于最新的原生平台功能。

构建服务限制

高级构建功能需要订阅付费计划，免费计划有构建时间和次数限制。

学习曲线

对于完全没有 React 或 JavaScript 经验的开发者，仍有一定的学习曲线。
