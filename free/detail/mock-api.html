What's a Mock API Anyway?

Imagine you're building a video game, but you don't have all the characters ready yet. A Mock API is like creating pretend characters to use while you build!

It's a fake database that programmers use to test their apps before connecting to the real thing. It helps you:

  • Build apps even when the real data isn't ready
  • Test how your app works with different information
  • Learn how to connect programs together

How Does It Work?

1

Create Fake Data

Make up information like a list of your favorite games or characters.

2

Set Up Your Mock API

Use special tools to turn your fake data into a pretend database.

3

Build and Test

Use your Mock API to make sure your app works perfectly!

Let's Try an Example!

Let's create a Mock API with a list of your favorite games. Here's how it might look: 

{
  "games": [
    {
      "id": 1,
      "name": "Adventure Quest",
      "type": "RPG",
      "rating": 5
    },
    {
      "id": 2,
      "name": "Race Masters",
      "type": "Racing",
      "rating": 4
    },
    {
      "id": 3,
      "name": "Puzzle World",
      "type": "Puzzle",
      "rating": 5
    }
  ]
}

Now we can get information from our Mock API. Let's try to get all the game names: 

// This code gets games from our Mock API
fetch('https://our-mock-api.com/games')
  .then(response => response.json())
  .then(data => {
    // Show all game names
    data.games.forEach(game => {
      console.log(game.name);
    });
  });
