What Are GitHub Actions Anyway?

Imagine you have a team of tiny helpers that work for you whenever you make changes to your code. That's what GitHub Actions are!

Automation Helpers

They can automatically check your code for mistakes whenever you make changes. Like having a teacher that checks your homework instantly!

Do Cool Stuff

They can build games, websites, or apps automatically. It's like having a factory that makes your creations for you!

Share Your Work

They can help share your projects with the world automatically. Like sending your art to a gallery without leaving your house!

How Do GitHub Actions Work?

It's like setting up a treasure hunt for your code! You create a list of steps, and GitHub Actions follows them.

1. You Create a "Workflow"

This is like writing a recipe that tells GitHub Actions what to do. It's a special file with instructions.

1
.github/workflows/my-first-action.yml
name: My First Action
on: push

jobs:
  hello:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Hello, World!"
2

2. Write Your Instructions

You tell it when to run (like when you "push" code) and what to do (like saying "Hello, World!").

3. Watch It Run!

GitHub will automatically run your workflow when the time comes. It's like magic - your instructions come to life!

3

Try It Yourself!

Let's see what GitHub Actions can do with a simple example.

Automatic Math Checker

This GitHub Action checks if your math is correct. Try changing the numbers!

math-checker.yml 
name: Check Math
on: push

jobs:
  check-sum:
    runs-on: ubuntu-latest
    steps:
      - name: Check 2 + 3
        run: |
          if [ $(( + )) -eq 5 ]; then
            echo "Math is correct!"
            exit 0
          else
            echo "Oops, that's wrong!"
            exit 1
          fi
