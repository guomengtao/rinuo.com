What is Vitest?

Vitest is a Vite-native test runner that combines the best features of Jest and Vue Test Utils. It’s designed to be fast, flexible, and easy to integrate with modern frontend workflows.

Blazing Fast

Leverages Vite’s esbuild-based bundling to deliver instant test feedback—no more waiting for slow builds.

Seamless Integration

Works out of the box with Vite projects, supporting Vue, React, Svelte, and other popular frameworks.

TypeScript First

Built with TypeScript, offering full type support for test files and configurations—no extra setup needed.

Founder & Project Origin

Evan You & The Vite Team

Vitest was created by Evan You (the creator of Vue.js) and maintained by the core Vite team. Evan’s vision was to build a test runner that matched Vite’s speed and developer experience.

Before Vitest, Vite users relied on Jest or Mocha—tools not optimized for Vite’s esbuild pipeline. This created friction in development workflows, especially for large projects.

Project Origin Story

Vitest started as an experimental side project in 2021, born from the need to solve a specific pain point: slow test execution in Vite-based projects.

The first public release (v0.1.0) launched in October 2021, and it quickly gained traction in the Vite community. By 2023, Vitest became the de facto test runner for Vite projects, with contributions from over 200 developers worldwide.

Pros & Cons

A balanced look at what makes Vitest great—and where it’s still growing.

Pros

  • Blazing fast test execution (2–10x faster than Jest for Vite projects)
  • Seamless Vite integration—no extra configuration for Vite projects
  • Native TypeScript support with zero setup
  • Rich plugin ecosystem (Vue, React, Svelte, MSW, etc.)
  • Modern features like hot module replacement for tests

Cons

  • Smaller community than Jest—fewer third-party tutorials/resources
  • Less mature for non-Vite projects (works but not optimized)
  • Some Jest APIs require adaptation (not 100% drop-in replacement)
  • Documentation is comprehensive but less polished than Jest’s
  • Limited support for Node.js backend testing (focused on frontend)

Get Started with Vitest

Follow these simple steps to add Vitest to your project.

  1. 1

    Set Up a Vite Project (if you don’t have one)

    If you’re starting from scratch, create a new Vite project first:

    npm create vite@latest my-vitest-project -- --template vue
    cd my-vitest-project
    npm install
  2. 2

    Install Vitest

    Install Vitest as a development dependency:

    npm install -D vitest
  3. 3

    Add a Test Script

    Update your package.json to include a test script:

    "scripts": {
    "test": "vitest",
    "test:watch": "vitest watch",
    "test:run": "vitest run"
    }
  4. 4

    Write Your First Test

    Create a test file (e.g., src/utils.test.ts) and write a simple test:

    import { expect, test } from 'vitest'
    import { add } from './utils'

    test('adds 1 + 2 to equal 3', () => {
    expect(add(1, 2)).toBe(3)
    })
  5. 5

    Run Your Tests

    Start the test runner:

    npm test

    You’ll see your test pass in seconds—thanks to Vitest’s speed!
