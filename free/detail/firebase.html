Firebase 的缺点

虽然 Firebase 功能强大，但也存在一些局限性需要了解

替代方案比较

Firebase vs AWS Amplify 更适合小型项目
Firebase vs Parse 更易上手但灵活性较低
Firebase vs Supabase PostgreSQL vs NoSQL

扩展性成本高

当应用规模增长超出免费额度后，Firebase 的成本可能快速增加。特别是数据库操作和存储，对于大规模应用可能比自托管解决方案昂贵得多。

数据库限制

Firestore 和 Realtime Database 都有查询限制，不支持复杂的 SQL 操作。对于需要复杂查询和事务的应用，可能会遇到瓶颈。

供应商锁定

迁移出 Firebase 可能比较困难，特别是使用了其专有功能（如特定的认证方法、云函数触发器）的应用，转换成本较高。

地区限制

Firebase 服务在某些地区不可用，且数据存储位置选择有限。对于有特定数据本地化要求的应用可能不合适。

自定义功能限制

虽然提供了云函数，但与完全控制的服务器环境相比，自定义能力仍然有限。某些高级后端功能实现起来较为复杂。

停机风险

作为托管服务，依赖于 Google 的基础设施。虽然可靠性高，但历史上也发生过几次重大 outage，可能影响应用可用性。
